---
title: "Getting Started"
description: "Define a tool and run it through the runtime in a few minutes"
---

The toolkit is a set of small, independent packages — install only what you
need. To define and run a tool you need two.

<Steps>
  <Step title="Install the packages">
    Every package is published under the `@ai-application-toolkit/*` scope and
    ships as ESM with TypeScript types.

    ```bash
    pnpm add @ai-application-toolkit/tool @ai-application-toolkit/runtime
    ```
  </Step>

  <Step title="Define your first tool">
    A [Tool](/concepts/tool) is a typed, executable function described by a JSON
    Schema. The schema lets the runtime validate input before your code runs,
    and lets the same tool be handed to model providers and MCP unchanged.

    ```ts
    import { defineTool } from '@ai-application-toolkit/tool'

    const hello = defineTool({
      id: 'hello',
      description: 'Greet someone by name',
      input: {
        type: 'object',
        properties: { name: { type: 'string' } },
        required: ['name'],
        additionalProperties: false
      },
      execute: async (input: { name: string }) => {
        return { message: `Hello, ${input.name}` }
      }
    })
    ```
  </Step>

  <Step title="Run it through the runtime">
    The [Runtime](/concepts/runtime) executes tools by id. It validates the
    input against the schema, enforces timeouts, normalizes errors, and emits
    trace events.

    ```ts
    import { createRuntime } from '@ai-application-toolkit/runtime'

    const runtime = createRuntime({ tools: [hello] })

    const result = await runtime.executeTool({
      toolId: 'hello',
      input: { name: 'AI Toolkit' }
    })

    console.log(result) // { message: 'Hello, AI Toolkit' }
    ```
  </Step>
</Steps>

<Note>
  If you pass input that doesn't match the schema (e.g. a missing `name`), the
  runtime throws a `ToolkitError` with code `TOOL_INPUT_INVALID` before
  `execute` is ever called.
</Note>
